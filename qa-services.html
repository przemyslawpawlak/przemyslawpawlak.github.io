01 / COVERAGE

What gets tested

Four engagement types, one standard of coverage. Every report ships with severity ratings, full reproduction steps, and a complete account of what was — and wasn't — found.

Available Web & SaaS

Most bugs don't live on the happy path. After 20+ years of professional QA, I test the way experienced users actually behave — different browsers, resolutions, window sizes, unexpected input sequences, and every detour away from the obvious flow before I ever take it.

What you get is a professional bug report you can act on immediately. Every issue documented with clear reproduction steps, severity rating, environment details, and expected versus actual results. Coverage summary included so you know exactly what was tested and what wasn't. Any issue that couldn't be reproduced gets a full account of everything attempted — you'll never wonder if something was just missed.

  • Cross-browser behavior
  • Multiple resolutions & window sizes
  • Form & input validation
  • User flow & navigation
  • Boundary & edge-case input
  • Session & login behavior
Best for: Web applications, SaaS tools, e-commerce stores, client portals, booking systems.
Available Indie Game

Games break in places developers never look because developers know exactly how their game is supposed to be played. I interact with it differently — exhausting every available input, every combination of actions, every detour from the intended path before I ever follow a waypoint. And as player abilities and mechanics unlock through progression, the test surface expands — I update my approach accordingly rather than assuming the early game inputs are the only ones that matter.

Beyond gameplay, the things that kill a launch get priority attention. Crashes, memory issues, frame rate degradation under load, and anything that makes the game flat out unplayable get flagged first. A game with rough edges ships. A game that crashes on startup doesn't.

  • Full input & control coverage
  • Progression-based ability unlocks
  • Crash & stability testing
  • Frame rate under load
  • Save / load integrity
  • Edge-case player behavior
For efficient and thorough testing, game engagements require access to debug tools, cheats, or a dev build with level and checkpoint skips. This allows full coverage of content without unnecessary time spent on legitimate playthrough progress, keeping your engagement focused and on schedule.
Best for: PC indie games, Steam Early Access releases, pre-launch readiness checks, Unity and Godot projects.
Available Desktop (Windows)

Desktop apps have a deceptively wide test surface. Every field has limits, every button has expected behavior, every function has edge cases — and without a complete design document covering all of it, testing is guesswork. I work from documentation where it exists and flag the gaps where it doesn't. That gap itself is often where the most interesting bugs live.

The same philosophy applies here as everywhere else — the happy path gets tested last. Input boundaries, unexpected sequences, window resizing, multiple monitor configurations, installation and uninstall behavior — everything that a developer focused on core functionality tends to leave until it's too late.

  • Field limits & input boundaries
  • Button & function behavior
  • Window resizing / multi-monitor
  • Install & uninstall flow
  • Design doc gap flagging
  • Unexpected input sequences
Best for: Windows utilities, productivity tools, niche business software, indie desktop applications.
Android Only Mobile

Android testing available for indie developers and small teams targeting the Google Play market. Testing performed on real Android hardware plus emulator coverage across multiple API levels and screen sizes to catch device-specific behavior that emulators alone won't surface.

Same deliverable standard applies — severity-rated bug report, full reproduction steps, coverage summary, and a complete account of anything that resisted reproduction.

  • Real device + emulator coverage
  • Multiple API levels & screen sizes
  • Touch input edge cases
  • Performance under load
  • Install & permissions flow
  • Orientation & interruption handling
iOS testing not currently available. Android only.
Best for: Android utilities, indie mobile games, early access and beta builds targeting Google Play.
